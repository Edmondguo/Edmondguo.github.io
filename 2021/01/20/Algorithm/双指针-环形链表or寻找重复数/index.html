环形链表&寻找重复数

环形链表

给定一个链表，判断链表中是否有环。

如果链表中有某个节点，可以通过连续跟踪 next 指针再次到达，则链表中存在环。 为了表示给定链表中的环，我们使用整数 pos 来表示链表尾连接到链表中的位置（索引从 0 开始）。 如果 pos 是 -1，则在该链表中没有环。注意：pos 不作为参数进行传递，仅仅是为了标识链表的实际情况。

如果链表中存在环，则返回 true 。 否则，返回 false 。

最后一个return False是为了长度为1的情况。

https://leetcode-cn.com/problems/linked-list-cycle/

环形链表II

给定一个链表，返回链表开始入环的第一个节点。 如果链表无环，则返回 null。

为了表示给定链表中的环，我们使用整数 pos 来表示链表尾连接到链表中的位置（索引从 0 开始）。 如果 pos 是 -1，则在该链表中没有环。注意，pos 仅仅是用于标识环的情况，并不会作为参数传递到函数中。

说明：不允许修改给定的链表。

进阶：你是否可以使用 O(1) 空间解决此题？

思路：1. 先用双指针判断是否有环；2. 找到环后，指针在环内部，再用双指针算出环的长度；3. 重新建立从头开始的双指针，快指针走环的长度步，然后一起走，相遇的地方就是环的入口。

优化：1. 先用双指针判断是否有环；2. 找到环后，head和slow一起走，走到相遇，就是入口。

推导：设开头长a，换长b，f为快指针长度，s为慢指针长度。找到环时，f = s+ nb，f = 2s，推出s=nb。head走a步到达入口，s走a步a+nb，正好也到达入口。

https://leetcode-cn.com/problems/linked-list-cycle-ii/

寻找重复数

给定一个包含 n + 1 个整数的数组 nums ，其数字都在 1 到 n 之间（包括 1 和 n），可知至少存在一个重复的整数。

假设 nums 只有 一个重复的整数 ，找出 这个重复的数 。

如何证明 nums 中至少存在一个重复的数字?
你可以在不修改数组 nums 的情况下解决这个问题吗？
你可以只用常量级 O(1) 的额外空间解决这个问题吗？
你可以设计一个时间复杂度小于 O(n2) 的解决方案吗？

思路：重复的数不一定只重复一次，所以数组中重复的数字一题的哈希思想的方法不能用了。

把下标index和nums映射，如果有重复，那么就存在多对一的映射，如果把映射关系看成链表，找重复就是找环入口

https://leetcode-cn.com/problems/find-the-duplicate-number/
