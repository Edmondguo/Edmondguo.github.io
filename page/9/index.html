旋转数组的最小数字

把一个数组最开始的若干个元素搬到数组的末尾，我们称之为数组的旋转。输入一个递增排序的数组的一个旋转，输出旋转数组的最小元素。例如，数组 [3,4,5,1,2] 为 [1,2,3,4,5] 的一个旋转，该数组的最小值为1。 注意数组中可能存在重复的元素。

二分法：中点向下取整，和右端点比较。如果中点大则更新head=mid+1；如果中点小则更新tail=mid；如果相等则tail-=1

相等时回缩右端点：如果结果<tail，回缩没问题；如果结果=tail，回缩虽然越过了x，但是最终返回结果的值是正确的的。

为什么不和左端点比较：二分是为了判断mid在哪个排序数组中，从而缩小区间。在只有右排序数组的情况下，找到比左端点大的mid其实他是在右排序数组。

寻找两个正序数组的中位

给定两个大小为 m 和 n 的正序（从小到大）数组 nums1 和 nums2。请你找出并返回这两个正序数组的中位数。

进阶：你能设计一个时间复杂度为 O(log (m+n)) 的算法解决此问题吗？

思路：二分法，先实现找两个数组的第k小的数，可以比较两个数列第k//2的数的大小，小的那个数列的那k//2个数必然比k小。这样就排除了k//2个数。然后在继续进行，直到两个数列其中一个为空，或者k==1。

需要注意可能数列比k//2小的情况。一种处理方式当出现这种情况时，两边都截断短数列的长度，不过这种情况可能导致时间复杂度退化成$O(m+n)$，比如[5],[1,2,3,4,5]这种情况。

最后合并奇偶的非常巧妙，求两次，看代码吧。

LRU 缓存机制

运用你所掌握的数据结构，设计和实现一个 LRU (最近最少使用) 缓存机制 。
实现 LRUCache 类：

LRUCache(int capacity) 以正整数作为容量 capacity 初始化 LRU 缓存
int get(int key) 如果关键字 key 存在于缓存中，则返回关键字的值，否则返回 -1 。
void put(int key, int value) 如果关键字已经存在，则变更其数据值；如果关键字不存在，则插入该组「关键字-值」。当缓存容量达到上限时，它应该在写入新数据之前删除最久未使用的数据值，从而为新的数据值留出空间。

进阶：你是否可以在 O(1) 时间复杂度内完成这两种操作？

思路：实现双向链表，存储key-value，另外通过哈希表存储key-链表节点。

鸡蛋掉落

你将获得 K 个鸡蛋，并可以使用一栋从 1 到 N 共有 N 层楼的建筑。

每个蛋的功能都是一样的，如果一个蛋碎了，你就不能再把它掉下去。

你知道存在楼层 F ，满足 0 <= F <= N 任何从高于 F 的楼层落下的鸡蛋都会碎，从 F 楼层或比它低的楼层落下的鸡蛋都不会破。

每次移动，你可以取一个鸡蛋（如果你有完整的鸡蛋）并把它从任一楼层 X 扔下（满足 1 <= X <= N）。

你的目标是确切地知道 F 的值是多少。

无论 F 的初始值如何，你确定 F 的值的最小移动次数是多少？

思路：大小为K+1 * N+1的dp数组。初始化0,1行列

状态转移方程：$dp[k][n] = min{max{dp[k-1][m-1],dp[k][n-m]}+1, where\ m \in[1,n] }$

上述方法是超时的，原因是最内层循环$dp[k-1][m-1],dp[k][n-m]$随着m的增大分别是单调递增和单调递减的。所以他们的max函数随着m增大一定是先递减后递增的。所以可以优化为二分法找最小。

青蛙过河

一只青蛙想要过河。 假定河流被等分为 x 个单元格，并且在每一个单元格内都有可能放有一石子（也有可能没有）。 青蛙可以跳上石头，但是不可以跳入水中。

给定石子的位置列表（用单元格序号升序表示）， 请判定青蛙能否成功过河（即能否在最后一步跳至最后一个石子上）。 开始时， 青蛙默认已站在第一个石子上，并可以假定它第一步只能跳跃一个单位（即只能从单元格1跳至单元格2）。

如果青蛙上一步跳跃了 k 个单位，那么它接下来的跳跃距离只能选择为 k - 1、k 或 k + 1个单位。 另请注意，青蛙只能向前方（终点的方向）跳跃。

请注意：

  • 石子的数量 ≥ 2 且 < 1100；
  • 每一个石子的位置序号都是一个非负整数，且其 < 231；
  • 第一个石子的位置永远是0。

思路：DP哈希表dic[i]上维护一个set，里面存储的是所有能跳到这个石头的步数。

所以这个DP哈希表是当前的来更新后面的，对于dic[i]里的每一个步数，都计算步数-1，步数，步数+1所到达的石头是否合法，如果合法，就为哈希表里那个石头对应的set加上步数。计算到最后，最后一个石头上如果有步数，说明能达到。如果是空的，说明不能达到。

零钱兑换

给定不同面额的硬币 coins 和一个总金额 amount。编写一个函数来计算可以凑成总金额所需的最少的硬币个数。如果没有任何一种硬币组合能组成总金额，返回 -1。

你可以认为每种硬币的数量是无限的。

DP

其实等价于完全背包问题，只不过背包问题的表述是装满背包的情况下价值最高是多少；零钱兑换可以把“价值”看做“钱的个数”（也就是1），把“体积”看做是“钱的面值”。所以装满就是面值达到总金额，要求硬币个数最少就是要求价值最少。

另一种容易思考的写法：DP数组中存储当前金额上最小的硬币个数，dp[i]的计算通过循环coins 查看符合条件的dp[i-coin]中的最小的

DFS

首先对coins倒排序，保证最大的先遍历

维护一个全局变量ans，用来已经满足的组合中的最小值。

递归结束条件：

  • 剩余的amount==0
  • index超出上限

剪枝条件：

  • 能被当前的硬币整除（因为硬币是从大到小排的，所以能被当前的硬币整除，后面就没有遍历的必要了）
  • 循环进入dfs前，一定要判断要进入的count是否已经大于min_record，如果大于，再递归已经没有意义了
