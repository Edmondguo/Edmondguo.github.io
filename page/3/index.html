K 个一组翻转链表

给你一个链表，每 k 个节点一组进行翻转，请你返回翻转后的链表。

k 是一个正整数，它的值小于或等于链表的长度。

如果节点总数不是 k 的整数倍，那么请将最后剩余的节点保持原有顺序。

栈

用栈实现，缺点是$O(k)$的空间复杂度

四指针

把链表按照k个分成一组，使用一个指针每次向前移动k步，然后翻转这部分链表，将翻转后的链表和原链表头尾相接。

在过程中需要维护head指针的上一个节点pre指针、子链表的头尾指针head、tail和tail的下一个指针nex。

https://leetcode-cn.com/problems/reverse-nodes-in-k-group/

递增的三元子序列

给定一个未排序的数组，判断这个数组中是否存在长度为 3 的递增子序列。

数学表达式如下:

如果存在这样的 i, j, k, 且满足 0 ≤ i < j < k ≤ n-1，
使得 arr[i] < arr[j] < arr[k] ，返回 true ; 否则返回 false 。
说明: 要求算法的时间复杂度为 O(n)，空间复杂度为 O(1) 。

解析：遍历一遍数组，希望遍历到的这个数three，前面已经有一个比他小的数two，再前面有一个比two小的数one。
我们需要维护两个变量：one和two。代表递增子序列的第一个数和第二个数。

https://leetcode-cn.com/problems/increasing-triplet-subsequence/solution/pou-xi-ben-zhi-yi-wen-bang-ni-kan-qing-t-3ye2/

https://leetcode-cn.com/problems/increasing-triplet-subsequence/

滑动窗口的最大值

给定一个数组 nums 和滑动窗口的大小 k，请找出所有滑动窗口里的最大值。

思路：由于每次滑动窗口，左边都可能弹出滑动窗口的最大值，所以类似min函数的栈一样，维护了一个存储非严格降序的双端队列。里面只存储可能成为最大值的数字的index，所以每次向右滑动时，都要将右端小于它的统统移除，以维持非严格降序的特性。当队首元素index和当前元素插值大于k-1时，要移除队首元素。

https://leetcode-cn.com/problems/hua-dong-chuang-kou-de-zui-da-zhi-lcof/

https://leetcode-cn.com/problems/sliding-window-maximum/
