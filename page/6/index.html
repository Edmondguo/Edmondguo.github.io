回文对

给定一组 互不相同 的单词， 找出所有不同 的索引对(i, j)，使得列表中的两个单词， words[i] + words[j] ，可拼接成回文串。

思路：对于一个字符串，如果能和其他的构成回文，那么他可以被拆分为子串+回文或者回文+子串，其中子串是其他字符串的颠倒。所以需要一个哈希表来储存字符串的颠倒。

需要注意的是许多边界条件。子串需要涵盖整个字符串。

https://leetcode-cn.com/problems/palindrome-pairs/

部分排序

给定一个整数数组，编写一个函数，找出索引m和n，只要将索引区间[m,n]的元素排好序，整个数组就是有序的。注意：n-m尽量最小，也就是说，找出符合条件的最短序列。函数返回值为[m,n]，若不存在这样的m和n（例如整个数组是有序的），请返回[-1,-1]。

思路：感觉只能背下来。

从左往右遍历，记录当前为止最大的值。如果当前值比最大值大，则更新最大值；如果当前值比最大值小，则更新last为当前值的索引。

从右往遍历，记录当前为止最小的值。如果当前值比最小值小，则更新最小值；如果当前值比最小值大，则更新first为当前值的索引。

所以最后获得的last是左边有比他大的值的最靠右的索引；first是右边有比他小的值的最靠左的索引。

https://leetcode-cn.com/problems/sub-sort-lcci/

移动零&移除元素&调整数组顺序使奇数位于偶数前面

移动零

给定一个数组 nums，编写一个函数将所有 0 移动到数组的末尾，同时保持非零元素的相对顺序。

思路：使用快慢双指针，左指针指向当前已经处理好的序列的尾部，右指针指向待处理序列的头部。

右指针不断向右移动，每次右指针指向非零数，则将左右指针对应的数交换，同时左指针右移。

注意到以下性质：

左指针左边均为非零数；

右指针左边直到左指针处均为零。

因此每次交换，都是将左指针的零与右指针的非零数交换，且非零数的相对顺序并未改变。

https://leetcode-cn.com/problems/move-zeroes/

盛最多水的容器

给你 n 个非负整数 $a_1，a_2，…，a_n$，每个数代表坐标中的一个点 $(i, a_i)$ 。在坐标内画 n 条垂直线，垂直线 i 的两个端点分别为 $(i, a_i)$ 和 $(i, 0)$ 。找出其中的两条线，使得它们与 x 轴共同构成的容器可以容纳最多的水。

思路：面积取决于较短的那个长度，首尾各一个指针，记录组成容器的最大值，并每次将短的指针向内移动一位。每次指针移动都是为了排除一根柱子，这根柱子在的情况下不可能达到最优解。

https://leetcode-cn.com/problems/container-with-most-water/

环形链表&寻找重复数

环形链表

给定一个链表，判断链表中是否有环。

如果链表中有某个节点，可以通过连续跟踪 next 指针再次到达，则链表中存在环。 为了表示给定链表中的环，我们使用整数 pos 来表示链表尾连接到链表中的位置（索引从 0 开始）。 如果 pos 是 -1，则在该链表中没有环。注意：pos 不作为参数进行传递，仅仅是为了标识链表的实际情况。

如果链表中存在环，则返回 true 。 否则，返回 false 。

最后一个return False是为了长度为1的情况。

https://leetcode-cn.com/problems/linked-list-cycle/

环形链表II

给定一个链表，返回链表开始入环的第一个节点。 如果链表无环，则返回 null。

为了表示给定链表中的环，我们使用整数 pos 来表示链表尾连接到链表中的位置（索引从 0 开始）。 如果 pos 是 -1，则在该链表中没有环。注意，pos 仅仅是用于标识环的情况，并不会作为参数传递到函数中。

说明：不允许修改给定的链表。

进阶：你是否可以使用 O(1) 空间解决此题？

思路：1. 先用双指针判断是否有环；2. 找到环后，指针在环内部，再用双指针算出环的长度；3. 重新建立从头开始的双指针，快指针走环的长度步，然后一起走，相遇的地方就是环的入口。

优化：1. 先用双指针判断是否有环；2. 找到环后，head和slow一起走，走到相遇，就是入口。

推导：设开头长a，换长b，f为快指针长度，s为慢指针长度。找到环时，f = s+ nb，f = 2s，推出s=nb。head走a步到达入口，s走a步a+nb，正好也到达入口。

https://leetcode-cn.com/problems/linked-list-cycle-ii/

寻找重复数

给定一个包含 n + 1 个整数的数组 nums ，其数字都在 1 到 n 之间（包括 1 和 n），可知至少存在一个重复的整数。

假设 nums 只有 一个重复的整数 ，找出 这个重复的数 。

如何证明 nums 中至少存在一个重复的数字?
你可以在不修改数组 nums 的情况下解决这个问题吗？
你可以只用常量级 O(1) 的额外空间解决这个问题吗？
你可以设计一个时间复杂度小于 O(n2) 的解决方案吗？

思路：重复的数不一定只重复一次，所以数组中重复的数字一题的哈希思想的方法不能用了。

把下标index和nums映射，如果有重复，那么就存在多对一的映射，如果把映射关系看成链表，找重复就是找环入口

https://leetcode-cn.com/problems/find-the-duplicate-number/

最小覆盖子串

给你一个字符串 s 、一个字符串 t 。返回 s 中涵盖 t 所有字符的最小子串。如果 s 中不存在涵盖 t 所有字符的子串，则返回空字符串 "" 。

注意：如果 s 中存在这样的子串，我们保证它是唯一的答案。

思路：滑动窗口法，双指针。先移动后指针直到包含t，然后移动前指针直到恰巧不包含t，然后再移动后指针，循环往复。

上面while all(map(lambda x: lookup[x]>=t[x] , t.keys()))判断是否包含耗时较大，可以构造一个counter=len(t)，当后指针的移动对包含t做出贡献时，counter递减1。通过counter是否为0来判断当前是否包含t。
